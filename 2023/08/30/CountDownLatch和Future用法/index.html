1. CountDownLatch 概念

CountDownLatch 可以用于控制线程的同步等待。CountDownLatch 通常用于一个或多个线程等待其他线程的操作完成后再继续执行。

CountDownLatch 可以用于很多场景，例如等待多个子任务完成后再执行主任务，控制并发测试中的线程同步，或者等待多个服务初始化完成后再启动应用程序等。

CountDownLatch 定义了一个计数器，和一个阻塞队列， 当计数器的值递减为0之前，阻塞队列里面的线程处于挂起状态，直到所有线程执行完毕。

要点：

  1. 初始化计数：CountDownLatch 的实例通过指定一个初始计数来初始化，该计数表示需要等待的线程数量。
  2. 线程等待：线程可以通过 await() 方法来等待计数器归零。当调用 await() 方法时，如果计数器不为零，则线程将进入等待状态，直到计数器变为零。
  3. 计数减少：通过 countDown() 方法可以将计数器减少1。每次调用 countDown() 方法都会使计数器减少1，直到计数器达到零。
  4. 等待阻塞：await() 方法调用后，当前线程会被阻塞，直到满足以下任一条件：计数器归零，或者当前线程被中断。
  5. 倒计时完成后：当计数器归零时，所有被阻塞的线程将被释放，可以继续执行后续操作。

示例代码：

public class CountDownLatchDemo {

    private static Map map = new HashMap<>();
    private static CountDownLatch countDownLatch = new CountDownLatch(2);

    public static void main(String[] args) {

        //记录开始时间
        long start = System.currentTimeMillis();

        Thread thread1 = new Thread(new Runnable() {
            @Override
            public void run() {
                try {
                    System.out.println("线程1任务开始执行");
                    Thread.sleep(6000);//任务执行需要6秒
                    map.put("thread1", 1);//保存结果值
                    countDownLatch.countDown();//标记已经完成一个任务
                    System.out.println("线程1任务执行完毕");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });
        thread1.start();

        Thread thread2 = new Thread(new Runnable() {
            @Override
            public void run() {
                try {
                    System.out.println("线程2任务开始执行");
                    Thread.sleep(3000);//任务执行需要3秒
                    map.put("thread2",2);//保存结果值
                    countDownLatch.countDown();//标记已经完成一个任务
                    System.out.println("线程2任务执行完毕");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });
        thread2.start();

        try {
            countDownLatch.await();//等待所有任务执行完毕
            long endTime = System.currentTimeMillis();//记录结束时间
            System.out.println("------统计所有线程全部完成--------");
            System.out.println("统计结果为：" + map.toString());
            System.out.println("任务总执行时间为" + (endTime - start) / 1000 + "秒");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

输出结果：

线程1任务开始执行
线程2任务开始执行
线程2任务执行完毕
线程1任务执行完毕
------统计所有线程全部完成--------
统计结果为：{thread1=1, thread2=2}
任务总执行时间为6秒

1.1 源码分析

1.1.1 创建计数器

当我们调用CountDownLatch countDownLatch=new CountDownLatch(2) 时候，此时会创建一个AQS的同步队列，并把创建CountDownLatch 传进来的计数器赋值给AQS队列的 state，所以state的值也代表CountDownLatch所剩余的计数次数；

public CountDownLatch(int count) {
     if (count < 0) throw new IllegalArgumentException("count < 0");
     this.sync = new Sync(count);//创建同步队列，并设置初始计数器值
 }

1.1.2 countDown()方法

调用 countDownLatch.countDown()方法，会对计数器进行减一，AQS（阻塞队列）内部是通过释放锁的方式，对state进行减1操作，当state=0的时候证明计数器已经递减完毕，此时会将AQS阻塞队列里的节点线程全部唤醒。

public void countDown() {
    this.sync.releaseShared(1); //Sync extends AbstractQueuedSynchronizer 
}

public final boolean releaseShared(int var1) {
    if (this.tryReleaseShared(var1)) {//当计数器为0时，
        this.doReleaseShared(); //唤醒线程
        return true;
    } else {
        return false;
    }
}

2. 线程池使用Future.get()

Future 它表示一个异步计算的结果。通过 Future 接口，我们可以提交任务给线程池进行异步执行，并在未来某个时间获取任务的执行结果。

future的核心思想是异步调用，当我们需要调用一个函数方法时，如果这个函数执行得很慢，那么我们就要进行等待。因此，我们可以让被调者立即返回，让它在后台慢慢处理这个请求。对于调用者来说，则可以先处理一些其他任务，在真正需要数据的场合再去尝试获得需要的数据。

要点：

  1. 提交任务：通过将任务提交给线程池的执行器（Executor）的 submit() 或 submit(Callable) 方法，可以获得一个 Future 对象来表示任务的执行。
  2. 异步执行：任务被线程池接受后，线程池会在后台使用一个线程或线程池中的多个线程来执行任务。任务的执行不会阻塞当前线程，可以继续进行其他操作。
  3. 获取结果：通过调用 Future 对象的 get() 方法，可以阻塞当前线程，直到任务执行完成并返回结果。get() 方法会返回任务的执行结果，如果任务还没有完成，则会阻塞等待。
  4. 取消任务：可以通过调用 Future 对象的 cancel(boolean) 方法来取消任务的执行。传递参数为 true 表示尝试取消正在执行的任务，传递参数为 false 表示不尝试取消正在执行的任务。
  5. 判断任务状态：可以通过调用 isDone() 方法来判断任务是否已经完成，或通过 isCancelled() 方法判断任务是否被取消。
  6. 阻塞超时：get() 方法还支持设置阻塞超时时间，如果任务在指定的时间内没有完成，则抛出超时异常。

示例代码

public class FutureDemo {

    public static void main(String[] args) {
        // 初始化线程池
        ExecutorService threadPool = Executors.newFixedThreadPool(5);

        // 初始化线程任务
        MyTask myTask = new MyTask(5);
        MyTask2 myTask2 = new MyTask2(5);

        // 向线程池递交线程任务
        Future<Integer> future = threadPool.submit(myTask);
        Future<Integer> future2 = threadPool.submit(myTask2);

        // 主线程休眠5秒，模拟主线程在做其他的事情
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        // 主线程获取异步任务的执行结果
        try {
            System.out.println("异步线程任务执行结果1 " + future.get());
            System.out.println("检查异步线程任务是否执行完毕1 " + future.isDone());
            System.out.println("异步线程任务执行结果2 " + future2.get());
            System.out.println("检查异步线程任务是否执行完毕2 " + future2.isDone());
        } catch (InterruptedException e) {
            e.printStackTrace();
        } catch (ExecutionException e) {
            e.printStackTrace();
        }
    }

    // 静态内部类，实现Callable接口的任务类
    static class MyTask implements Callable<Integer> {
        private int num;

        public MyTask(int num) {
            this.num = num;
        }

        @Override
        public Integer call() throws Exception {
            System.out.println(Thread.currentThread().getName() + " 正在执行异步任务");
            this.num += 1;
            return this.num;
        }
    }

    static class MyTask2 implements Callable<Integer> {
        private int num;

        public MyTask2(int num) {
            this.num = num;
        }

        @Override
        public Integer call() throws Exception {
            System.out.println(Thread.currentThread().getName() + " 正在执行异步任务");
            this.num *= 2;
            return this.num;
        }
    }
}

异步线程任务执行结果1 6

异步线程任务执行结果2 10
检查异步线程任务是否执行完毕2 true
